Module 01

What Is AI, Really?

Before you can master AI, you need to understand what it actually is — and what it isn't. Let's cut through the noise.

🧠

Large Language Models (LLMs)

LLMs are trained on massive text datasets and learn to predict the next word — billions of times over. This "simple" task produces a system that can reason, write code, summarize, translate, and much more. Claude, GPT, and Gemini are all LLMs.

🎯

Tokens & Context Windows

AI models don't read words — they read tokens (chunks of text). A context window is how much text a model can "hold in its head" at once. Claude's context window goes up to 200,000 tokens — enough for a full novel.

🌡️

Temperature & Parameters

Temperature controls creativity: low = precise and predictable, high = creative and exploratory. System prompts set the model's behavior and persona before your conversation begins. These levers shape every response.

🔗

The Transformer Architecture

Transformers use "attention mechanisms" to understand relationships between words across long distances. This is why AI understands that "it" in a complex sentence refers to something said 3 paragraphs earlier.

🛡️

RLHF & Safety Training

After pre-training, models are fine-tuned using human feedback (RLHF) to be helpful, harmless, and honest. Claude is built by Anthropic with a particular focus on Constitutional AI — embedding safety at the core.

⚡

Generative vs. Discriminative AI

Generative AI creates new content: text, images, code, audio. Discriminative AI classifies or judges existing content. Most modern AI assistants are generative — they produce, they don't just predict categories.

🔑 Key Insight: AI models don't "know" things the way you do. They're extraordinarily sophisticated pattern-matchers trained to produce useful text. Understanding this helps you write better prompts — you're not asking a search engine, and you're not talking to a human. You're collaborating with a probabilistic text model.

Module 03

The Art of Prompting

Prompting is a skill. The difference between a mediocre response and a brilliant one is almost always in how you ask. Learn the frameworks that work.

The RICE Framework

Essential

RICE is the simplest way to build complete prompts. Think of each element as giving the AI a different piece of what it needs to help you perfectly.

  • R
    Role Who should the AI act as? e.g. "You are a senior UX designer with 10 years at Google."
  • I
    Input What information are you giving it? Data, text, files, context — be explicit.
  • C
    Context Why does this matter? What's the situation, the constraints, the audience?
  • E
    Expectation What do you want back? Format, tone, length, style, structure — spell it out.
⭐ The better your RICE, the better the response. Every element you add narrows the solution space toward exactly what you need.

The STAR Framework

Advanced

For complex tasks where structure matters. Combine with a Persona for 80% of professional prompting scenarios.

  • S
    Situation The context and background — what's happening and why it matters.
  • T
    Task The specific action you want completed. Precise verbs: analyze, write, compare, extract.
  • A
    Action / Approach How you want it done — step-by-step, bullet points, devil's advocate, etc.
  • R
    Result The ideal output: format, length, tone, what "success" looks like.
📐 RICE Framework in practice — with prompt patterns like Few-shot, Chain-of-Thought, and Output Structuring
🎯 The 80/20 of Prompt Engineering — Persona + STAR gives you 80% effectiveness

Prompt Patterns That Work

🔢

Zero-shot

Straight request, no examples. Best for clear tasks.

📋

Few-shot

Give 2–3 examples to show the pattern you want.

🔗

Chain-of-Thought

Ask it to "think step by step" — dramatically improves reasoning.

🎭

Persona

Assign an expert role to unlock deeper domain knowledge.

📊

Output Structuring

Specify format: JSON, table, markdown, numbered list.

🔄

Iterative Refinement

Use follow-ups: "make it shorter", "add more data", "change tone to X".

Module 04B

The CLAUDE.md File

The single most powerful thing you can create for any Claude Code project. A CLAUDE.md file tells Claude everything it needs to know about your project — automatically, every session.

What is CLAUDE.md?

When Claude Code starts in any directory, it automatically reads CLAUDE.md (if it exists) and loads that context into every conversation. Think of it as a persistent memory file — your project's README for Claude. It never forgets, never needs re-explaining, and applies to every task in that workspace.

Where CLAUDE.md Lives

my-project/
  ├── src/
  │   ├── components/
  │   │   └── CLAUDE.md # component-level rules
  │   └── index.ts
  ├── CLAUDE.md # ← ROOT: auto-loaded always
  ├── package.json
  └── README.md
 
~/.claude/
  └── CLAUDE.md # global: applies everywhere
📂 Loading Order: Claude reads global (~/.claude/CLAUDE.md) first, then root project CLAUDE.md, then any sub-directory CLAUDE.md files. Each layer adds specificity. You can have project-wide rules AND component-specific rules.

A Real CLAUDE.md Example

# My SaaS Project — Claude Instructions ## Project Overview This is a Next.js 14 SaaS app using TypeScript, Prisma ORM, and Tailwind CSS. Deployed on Vercel. ## Tech Stack - Framework: Next.js 14 (App Router) - Database: PostgreSQL via Prisma - Auth: NextAuth.js v5 - Styling: Tailwind CSS + shadcn/ui - Tests: Vitest + Testing Library ## Code Conventions - Always use TypeScript strict mode - Components: PascalCase, files: kebab-case - Never use `any` type - Server actions go in /app/actions/ - Tests must cover happy path AND error paths ## What NOT to Do - Don't use pages/ router (we use app/) - Don't use class components - Don't commit .env files ## Commands npm run dev # start dev server npm run test # run test suite npm run db:push # push schema changes

Claude Instructions & System Prompts

Instructions are the layer between you and Claude's raw capability. Master them and you stop repeating yourself — Claude just knows how you work.

The Instruction Layers

Every Claude response is shaped by multiple layers of instruction, stacked in priority order. Understanding this hierarchy lets you control Claude precisely at the right level.

L1

Anthropic's Core Training

Claude's fundamental values, safety principles, and Constitutional AI guidelines. Baked in — cannot be overridden by any instruction.

Base
L2

Operator System Prompt

Set via API by developers building apps. Defines Claude's persona, capabilities, and constraints for that product. Applies to all users of that app.

Operator
L3

User Instructions / Preferences

What you set in claude.ai Settings → Profile. Applies across all your conversations. "Always respond in markdown", "I'm a senior engineer — skip basics".

User
L4

In-Conversation Instructions

What you say in the chat itself. Can temporarily override L3. Highest specificity — most recent instruction wins for that session.

Chat

Writing Effective System Instructions

🎭 Persona Instructions
You are a senior software architect with 15 years of experience in distributed systems. You value simplicity, correctness, and maintainability above all else. You're direct, opinionated, and always explain the "why".
📐 Format Instructions
Always structure responses with: 1) A one-line TL;DR, 2) The main answer in prose, 3) A "Watch Out For" section when relevant. Never use more than 3 levels of nesting in lists.
🎯 Scope Instructions
You help users of a B2B SaaS analytics platform. Only answer questions related to data analysis, SQL, Python, and BI tools. For off-topic questions, politely redirect to our documentation.
⚡ Behavior Instructions
Think step by step before answering complex problems. If you're uncertain, say so explicitly with a confidence percentage. Never make up facts — say "I don't know" when appropriate.
🌐 Context Instructions
My name is Priya. I'm a product manager at a fintech startup in Bangalore. I'm non-technical. Explain code concepts using analogies. My main priorities are user retention and reducing churn.

Module 04D

Context Engineering

Prompt engineering is what you ask. Context engineering is everything you set up before asking. The professionals live here.

The Context Window — What's Actually in It

Claude's context window (up to 200K tokens) holds everything simultaneously. How you fill it determines response quality. Every token costs attention — use them wisely.

Context Window — 200K tokens
Context Window Contents
🟠 System Prompt / Instructions~500–5K tokens
🔵 Documents & Files (PDFs, code, data)~10K–100K tokens
🟢 Conversation Historygrows over time
🟣 Tool Definitions (MCP, APIs)~1K–5K tokens
🟡 Your Current Messagevaries
⬜ Remaining Budget→ Claude's response
💡 Claude sees ALL of this simultaneously when generating its response

The 5 Principles of Context Engineering

🎯

1. Relevant Context Over Maximum Context

Don't dump everything you have. Curate. A focused 10-page document beats a 100-page one where 90% is irrelevant. Signal-to-noise ratio matters.

📍

2. Place Key Information Strategically

The most important instructions perform best at the beginning of the system prompt and at the end of the user message (recency effect). Critical rules go first.

🔄

3. Structure Beats Prose for Instructions

Numbered lists, headers, and clearly labeled sections outperform dense paragraphs for instructions. Claude parses structured text more reliably than extracting rules from prose.

🧹

4. Manage Context Across Long Conversations

In long conversations, earlier context gets "diluted". For complex multi-session work, use Projects. For very long chats, periodically summarize and restart with the summary as context.

✅

5. Examples Outperform Descriptions

Instead of describing the format you want, show one. A single well-crafted example in your context consumes ~200 tokens and returns 10x the formatting consistency.

Context Engineering Tactics

📄

Upload, Don't Describe

Instead of summarizing a document, attach it. Claude reads at ~100K tokens/second. Raw source material beats your paraphrase every time — no information lost in translation.

🗂️

Use Claude Projects

Projects in claude.ai persist documents and instructions across all sessions. Upload your codebase, brand guidelines, PRDs, and past decisions. Claude never "forgets" between conversations.

🔖

Reference, Don't Repeat

Once you've established context, refer back to it: "Using the architecture we discussed" or "Following the conventions in CLAUDE.md". Claude maintains full access to prior context in the window.

📊

Ground Decisions in Data

Attach raw data files (CSV, JSON) instead of summaries. Ask Claude to analyze the actual numbers. Context + data = AI that can make real decisions, not generic recommendations.

🧩

Chunk Large Tasks

For tasks larger than the context window, break into phases: "Phase 1: Research. Phase 2: Outline. Phase 3: Draft." Start each phase by summarizing what was decided in the last.

🪞

Ask Claude to Reflect Its Context

Before a big task, ask: "Summarize what you know about this project from the context you have." This surfaces gaps — anything missing is context you should add before proceeding.

🎓 The Expert's Mindset: Beginners write prompts. Intermediates engineer prompts. Experts engineer context. The highest-leverage work happens before you type your first question — setting up the right documents, the right instructions, and the right constraints so that every subsequent interaction is higher quality by default.
Context Engineering Challenge

Build a Fully-Loaded Claude Workspace

  1. Create a Project in claude.ai for a real project you're working on.
  2. Write a system instruction covering: your role, the project goal, your tech stack, and your preferred response format.

Module 06

10 Best Practices Every AI User Must Know

Separate yourself from the 99% who use AI casually. These are the habits of power users.

01

Always Give a Role

Setting expertise context unlocks dramatically deeper responses. AI calibrates its vocabulary, assumptions, and depth to the role you assign.

✓ "You are a senior financial analyst..."
✗ "Explain financial ratios"
02

Specify Output Format

Tell the model exactly how you want the result structured. Unformatted outputs require post-processing; formatted outputs are ready to use.

✓ "Return as a JSON array with keys: name, value, insight"
✗ "Give me a list"
03

Add Confidence Constraints

Tell the model to only share insights when it's >90% confident. This dramatically reduces hallucinations and keeps outputs trustworthy.

✓ "Only state things you're >90% confident about"
✗ Accepting all outputs uncritically
04

Iterate, Don't Restart

A conversation is a collaboration. Use follow-ups: "Make it punchier", "Add a counterargument", "Rewrite paragraph 2 for a CEO audience".

✓ Iterate → Refine → Elevate
✗ Starting over each time
05

Use Chain-of-Thought

Adding "think step by step" or "reason through this carefully" improves accuracy on logical, analytical, and mathematical tasks by 40–60%.

✓ "Think step by step before giving your answer"
✗ Asking for direct answers to complex problems
06

Attach Files & Context

Claude can read PDFs, CSVs, images, code files, and web content. Don't describe data in prose — attach it. More context = better analysis.

✓ Attach the spreadsheet, the brief, the document
✗ Copy-pasting scattered data manually
07

Anti-Sycophancy Instructions

Explicitly ask Claude to be honest and critical. "Don't be a sycophant. Tell me what's weak. Be sharper than comfortable." This unlocks real feedback.

✓ "Be honest, critical, and don't flatter me"
✗ Accepting the first polite answer
08

Use System Prompts for Consistency

In the API or Projects, set a system prompt that establishes your persona, writing style, and constraints so you don't repeat yourself every conversation.

✓ Save reusable context in Projects
✗ Re-explaining your context every session
09

Verify Critical Information

AI can hallucinate. For facts, numbers, citations, and statistics — always verify with primary sources. AI is a co-pilot, not an oracle.

✓ Cross-check facts with authoritative sources
✗ Publishing AI output without review
10

Think in Workflows, Not Queries

The biggest gains come from chaining AI into multi-step workflows: research → analyze → draft → refine → format. Map your processes, then find the AI bottlenecks.

✓ Design end-to-end AI-powered workflows
✗ Using AI for isolated one-off tasks only
